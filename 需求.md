# Role: 桌面端应用架构师与全栈开发专家 (Go + React + Wails)

## 项目背景
我们需要开发一款用于技术演示的高性能全屏画图软件（屏幕批注工具）。
核心诉求是：软件必须能像一块“透明玻璃”一样覆盖在整个屏幕上。用户通过**双击左侧 Ctrl 键**，可以在“画图模式”（拦截鼠标进行绘制）和“编码模式”（鼠标事件穿透，直接操作底层 IDE 或终端）之间无缝切换。

## 技术栈选型
* **核心框架:** Wails v2
* **后端:** Go (处理原生窗口属性、全局键盘 Hook、进程间通信 IPC)
* **前端:** React + JavaScript (纯 JS，不使用 TypeScript) + TailwindCSS
* **图形渲染:** HTML5 Canvas

## 核心架构与技术难点说明
1. **全屏透明窗口:** Go 端的 `wails.Run` 配置必须设置为：无边框 (`Frameless: true`)、置顶 (`AlwaysOnTop: true`)、背景全透明 (`BackgroundColour: &options.RGBA{R: 0, G: 0, B: 0, A: 0}`)。
2. **极简交互 (双击 Ctrl 切换模式):** * Go 端需要使用底层的系统 Hook（例如直接调用 Windows API 或使用成熟的 hook 库）。
   * 监听左侧 `Ctrl` 键。在 Go 端维护一个时间戳记录，如果连续两次按下 `Ctrl` 键的时间间隔小于 300ms，则翻转当前的状态（在“画图模式”和“穿透模式”之间切换）。
3. **事件穿透 (Click-through) 机制:**
   * **编码穿透模式:** Go 调用操作系统底层 API（如 Windows 的 `SetWindowLong` 注入 `WS_EX_TRANSPARENT`），使整个 Wails 窗口忽略鼠标事件，点击穿透到底层桌面。并通知前端隐藏画图工具栏。
   * **画图模式:** Go 恢复窗口的鼠标接收能力，前端 React 显示工具栏，并在 Canvas 上接收鼠标拖拽事件进行绘制。

## 执行计划：请严格按照以下阶段逐步实现，每次只完成一个 Phase

### Phase 1: 核心环境与“透明玻璃” MVP (当前阶段)
目标：跑通纯 JS 的 React 渲染和透明窗口。
1. 提供 Wails v2 初始化 React+JS 项目的命令。
2. 配置 Go 端的 `main.go`，成功唤起一个全屏、置顶且完全透明的窗口。
3. 在前端实现一个全屏的 Canvas 画板。
4. 实现基础的红线自由绘制功能（响应鼠标按下、移动、抬起事件）。

### Phase 2: 双击 Ctrl 监听与鼠标穿透 (等 Phase 1 验收后进行)
目标：实现画图与底层桌面操作的无缝切换。
1. 在 Go 端实现全局键盘 Hook，精准识别“双击左侧 Ctrl”动作（间隔 < 300ms）。
2. 编写 Go 方法调用系统 API 实现窗口级鼠标事件穿透的动态开关。
3. 双击触发时，Go 切换穿透状态，并通过 `events.Emit` 通知前端 React 更新 UI 状态。

### Phase 3: 完善画板功能 (等 Phase 2 验收后进行)
目标：丰富教学工具。
1. 添加简单工具栏（支持切换颜色、清屏）。
2. 实现绘制轨迹的撤销 (Undo) 功能。
